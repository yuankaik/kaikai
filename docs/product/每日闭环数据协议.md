## 原则

所有输出必须来自结构化数据。禁止脚本直接手写散题并画进 PDF。

标准链路：

```text
classroom_brief + grading_result + mastery_state + learner_profile
  -> daily_decision
  -> practice_spec
  -> pdf / console / grading_template / progress_report
```

## mastery 状态机

```text
未识别
  -> 入门验证
  -> 待回炉
  -> 做对但需讲清
  -> 理解通过
  -> 稳定掌握
  -> 已毕业
```

状态规则：

- 做错：`待回炉`。
- 做对但未讲：`做对但需讲清`。
- 做对且费曼 pass：`理解通过`。
- 连续正确 2 次：`稳定掌握`。
- 连续正确 3 次且至少 1 次费曼 pass：`已毕业`。

## 生成顺序

1. 读取课堂反馈。
2. 读取最新批改。
3. 更新掌握度。
4. 生成 decision。
5. 生成 practice spec。
6. 验证题目答案。
7. 渲染 PDF。
8. 检查 PDF 字号、换行、页数、空白。
9. 创建或更新本次提交文件夹，提示用户放入照片和语音。
10. 刷新家庭教师操作台。
