# 真题试卷搜索归档

## 工作流程

1. 创建归档目录：`~/exams/{语文,数学,英语,学而思,新东方,精锐教育,口算心算,优秀作文}/`
2. **三四五年级优先**，其次一二六年级
3. delegate_task 并行三科搜索 + 教育品牌搜索
4. 每科至少 3 个 subagent 并行
5. 提取内容 → MD 文件 + 答案
6. 持续搜索，不要轻易放弃

## 年级优先级

| 优先级 | 年级 | 原因 |
|--------|------|------|
| 🔴 最高 | 三、四、五年级 | kai哥核心需求 |
| 🟡 中等 | 一、二、六年级 | 补充覆盖 |
| 🟢 低 | 初中预备 | 有余力再做 |

## 教育品牌

学而思 > 新东方 > 精锐教育 > 昂立/四季/乐课力/绿光

## 搜不到的应对

教育网站需要付费/登录时：
- 基于沪教版/统编版大纲自行编写
- 确保每份有题目+答案
- 作文/日记/书信类天然无答案，属正常

## 原创练习材料生成（基于特级教师方法论）

当用户需要的是"让孩子练起来"的实操材料，而非理论/试卷时：

1. **阅读理解练习**：基于丁有宽读写结合法，自创短文 + 题目 + 答案
   - 每篇文体轮换：记叙文→写景→写人→寓言→叙事→写物→科普→散文→童话→应用文
   - 每篇5题，覆盖6大必考题型（概括、解词、赏析、情感、联系实际、含义）
   - 每题标注分值，附详细参考答案 + 错题分析区

2. **写作练习卡**：基于武凤霞/乔亚梦分年级专题体系
   - 每专题含：任务说明 → 方法步骤 → 范文片段 → 自查清单
   - 附万能开头/结尾模板 + 急救技巧（五感凑字法等）

3. **每日训练计划**：基于于永正五重教学法
   - 周一阅读→周二写作→周三精读→周四作文→周五泛读→周末综合
   - 含家长配合要点 + 参考书单

4. **输出PDF**：使用 `chinese-pdf-generation` 技能生成可打印PDF
   - 前提：`apt-get install fonts-noto-cjk` + `pip install weasyprint`

## 扩展用法：教学方法/名师资料搜索

本技能同样适用于搜索**特级教师教学方法、名师实操、教育理念**等非试卷类教育资料：

1. 搜索策略：百度百科获取教师简介和代表作 → 当当网获取书籍目录结构 → 道客巴巴/原创力搜索教学实录全文
2. 并行搜索模板：将8位教师分成3组，每组3个 subagent 并行搜索
3. 常见反爬：百度搜索触发验证码、搜狗/360 需要验证、doc88 需登录看全文
4. 绕过策略：优先用百度百科（无验证）+ 当当网（无验证），最后才尝试 doc88/book118
5. 输出格式：MD 结构化文档，含简介、理念、方法、代表作、适合年级五部分
6. PDF 生成：`python3 scripts/md2pdf.py <input.md> <output.pdf>` 用 weasyprint 转 PDF（需 `pip install --break-system-packages weasyprint`）

详见 `references/chinese-teachers-methodology-guide.md` 和 `references/top-8-teachers-quickref.md`。

## 答案缺失修复

搜索提取的试卷常缺答案。修复方法：
```bash
# 找到缺答案的文件
find ~/shanghai-exams -name "*.md" | while read f; do
  grep -q "答案\|Answer\|answer\|写作要点\|总结" "$f" || echo "$f"
done

# 排除作文类（天然无答案）
# 对试卷类逐个补充参考答案
```

## 同步到桌面

```bash
cp -r ~/shanghai-exams/* /mnt/c/Users/admin/Desktop/shanghai-exams/
```

## 验证步骤

完成后必须验证：
```bash
# 检查缺答案的文件（排除作文/日记/范文）
find exams/ -name "*.md" | while read f; do
  grep -q "答案\|Answer\|answer\|写作要点\|总结" "$f" || echo "缺: $f"
done
```

## 注意事项

- **作文/日记/书信/读后感类文件天然不需要"答案"**，验证时跳过
- doc88/book118 提取的试卷经常缺答案，需手动补充
- 所有试卷类文件必须有题有答案，缺一不可
- 3/4/5 年级优先级最高（用户明确要求）
